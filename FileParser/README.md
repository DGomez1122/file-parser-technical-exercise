# File Parser – Technical Exercise

## Overview
This console application reads a pipe-delimited text file, parses customer and transaction data, and generates a structured output file.

The application was designed to be simple, readable, and portable, following basic .NET best practices.

---

## Input
- The input file must be located in the `Incoming` folder.
- File name: `InputData.txt`
- Fields are delimited using the `|` character.

---

## Output
- The output file is generated in the `Outgoing` folder.
- File name: `OutputData.txt`
- Two types of records are produced:
  - `CUSTOMER`
  - `DETAIL`
